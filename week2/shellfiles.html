3 Shell expansion I: Wildcard expansion

Wildcard Matches
* Any number of any character, including nothing
? Any single character
[] and [^] One or none (^) of the “character set” within the brackets

–

  • With the following files in a directory:

    # sample1_F.fastq.gz sample1_R.fastq.gz
# sample2_F.fastq.gz sample2_R.fastq.gz
# sample3_F.fastq.gz sample3_R.fastq.gz

  • To match both “sample1” files:

    ls sample1_?.fastq.gz
ls sample1*
ls sample1*fastq.gz

4 Shell expansion I: Wildcard expansion (cont.)

Wildcard Matches
* Any number of any character, including nothing
? Any single character
[] and [^] One or none (^) of the “character set” within the brackets

  • With the following files in a directory:

    # sample1_F.fastq.gz sample1_R.fastq.gz
# sample2_F.fastq.gz sample2_R.fastq.gz
# sample3_F.fastq.gz sample3_R.fastq.gz

  • To match only files with forward (“F”) reads:

    ls *F*
ls *F.fastq.gz

  • To match files for sample1 and sample2 using only a character class:

    ls sample[12]*   # Method 1: List all possible characters
ls sample[1-2]*  # Method 2: Use ranges - [0-9], [A-Z], [a-z].
ls sample[^3]*   # Method 3: *Exclude* something.
Warning

[] works on character ranges only: 0-9 works but 10-13 does not.

  • Wildcards are only used to match and expand to existing file names:
    this process is also referred to as “globbing”.
Why do we call globbing a type of shell expansion?

The expansion –to all matching file names– is done by the shell,
not by ls or another command you might be using wildcards with.

Therefore, ls will see the list of files after the expansion.

  • You can make use of this when using shell expansion to move or delete files – first check what your command will see, then execute:

    echo sample[12]*
# sample1_F.fastq.gz sample1_R.fastq.gz sample2_F.fastq.gz sample2_R.fastq.gz

rm sample[12]*
Note

Examples so far may seem trivial, but you can use these techniques to easily operate on selections among 100s or 1000s of files.

Warning

Don’t confuse wildcards with regular expressions!

Regular expressions are useful across the board when coding,
and we will … TOODO


5 Shell expansion II: Brace expansion

cd ../..    # to: zmays-snps

Wildcard expansion looks for corresponding files and expands to whichever files are present: can’t be used to create files or dirs.

Brace expansion, on the other hand, expands to whatever you tell it to:

  • Use .. to indicate ranges:

    #mkdir -p data/obs/2021-01-{01..31} # Numeric range works!

mkdir figs
#touch figs/fig-1{A..F}             # Character range works

  • Use a comma-separated list:

    #mkdir data/obs/treatment-{Kr,Df,Tr}_temp-{lo,med,hi}

What will happen in these two cases?

cd figs
ls
# fig-1A fig-1B fig-1C fig-1D fig-1E fig-1F
#ls fig-1[A-G]
#ls fig-1{A..G}
  • Wildcard expansion will ignore the missing “plot-G”:
#ls fig-1[A-G]
# fig-1A fig-1B fig-1C fig-1D fig-1E fig-1F
  • Brace expansion will complain about the missing “plot-G”:
#ls fig-1{A..G}
# ls: cannot access 'plot-G': No such file or directory
# fig-1A fig-1B fig-1C fig-1D fig-1E fig-1F

6 Shell expansion III: Command substitution

  • Command substitution allows you to store and pass the output of a command (or pipeline of commands) to another command.

    These two examples show how that can be useful, and that a pipe wouldn’t work:

    echo "I have $(ls *fastq | wc -l) fastq files."
# I have 200 fastq files.

mkdir ../../results_$(date +%F)
# results_2021-01-21

  • It’s often convenient to save the result in a variable:

    n_files=$(ls *fastq | wc -l)
echo "$n_files"
Note

Variables don’t always have to be quoted like above ($n_files vs "$n_files"), but it is good practice as failing to do so can lead to problems. (Also: a variable can be put inside braces: ${n_files}.)


7 Programmatically renaming files

  • There are many different ways to do this in the shell –
    admittedly none as easy as one might have hoped.
    Here, we’ll use the basename command and a for loop.

  • basename strips any dir name that may be present from a file name (path), and optionally, removes any suffix too:

    basename selection/sample001_R1_001.fastq
# sample001_R1_001.fastq

# Specify a suffix to remove:
basename selection/sample001_R1_001.fastq "_001.fastq"
# sample001_R1

for loops are a verbose method for tasks like renaming,
but are intuitive and good practice.

  • The following shows the basic syntax of a for loop, and how the variable that is looped over is assigned to oldname, but recalled as $oldname:

    for oldname in *.fastq; do
>   echo "Old name: $oldname"
> done
Note

Note how we are using globbing (wildcard file name matching) to define what will be looped over.

  • Next, we assign a new name for each file, and echo the names as
    a sanity check:

    for oldname in *.fastq; do
>   newname=$(basename "$oldname" _001.fastq).fq
>   echo "Old/new name: $oldname $newname"
> done

  • Finally, we add the actual mv command:

    for oldname in *.fastq; do
>   newname=$(basename "$oldname" _001.fastq).fq
>   echo "Old/new name: $oldname $newname"
>   mv "$oldname" "$newname"
> done


8 Treating data as read-only: Viewing and modifying file permissions

8.1 Showing file permissions

  • To show file permissions, use ls with the -l (long format) option.shows file permissions.

    (The command below also uses the -a option to show all files, including hidden ones, and -h shows file sizes in “human-readable” format.)

File permission notation in ls -l

8.2 Changing file permissions

This can be done in several ways with the chmod command:

# chmod <who>=<permission-to-set>`:
chmod a=rwx  # a: all / u: user / g: group / o: others
   
# chmod <who>+<permission-to-add>:
chmod a+r  
# chmod <who>-<permission-to-remove>:
chmod o-w
  • Using numbers:
Nr Permission
1 execute
2 write
4 read
5 r + x
6 r + w
7 r + w + x

Three numbers are needed to specify permissions for user, group, and others:

# rwx permissions for all:
chmod 777 file.txt
# rwx for me, r for others:
chmod 744 file.txt

9 File permissions – Making your data read-only

To make your raw data read-only:

chmod a=r data/raw/*       # *a*ll = *r*ead
  
chmod u-w data/raw/*       # Remove write permission for *u*ser
  
chmod 400 data/raw/*       # User: read // group & others: none
Warning

Permissions for directories
One tricky aspects of file permissions is that to list a directory’s content, you need execute permissions for the dir!

This is something to take into account when you want to grant others access to your project e.g. at OSC.

To set execute permissions only for dirs throughout a dir hierarchy:

chmod -R a+X my-project-dir  # Note the *uppercase* X

9.1 Changing file permissions: In practice

  • First, let’s look at the original permissions of the files:

    # Assuming you are still in week02/zmays-snps/data/seqs
ls -l

  • Then, limit permissions for all to read, and check what we did:

    chmod a=r *fastq*
ls -l

  • What happens when we try to remove write-protected files?

    rm *fastq*

rm -f *fastq*

    While you (the file owner) can still remove write-protected files, others really can’t read/write/execute files without appropriate permissions.


10 Bonus content

10.1 Special characters and escaping them

We saw that wildcards like * have a special meaning. What if we had terrible file names, and needed to match a literal *?

  • A backslash (“\”) can be used to “escape” the special meaning of the character that follows it:

    ls
# 'my bad*file'
rm my\ bad\*file  # Tab autocomplete will work and insert \'s

  • Quoting a string of characters will also escape special meaning of characters within them, with single quotes ('...') being more severe than double quotes ("...") — more on this later.

    rm 'my bad*file'
Note

Backslashes can also assign special meaning – e.g. \t (tab) and \n (newline). We’ll talk more about these regular expressions later.

10.2 Using files across projects: Using symbolic links

Single files

  • A symbolic (or soft) links only links to the path of the original file, whereas a hard link directly links to the contents of the original file.

    Modifying a file via either a hard or soft link will modify the original file.

  • Create a symlink to a file using ln -s <source-file> [<link-name>]:

    # Only provide source: create link of the same name in the wd:
ln -s ~/share/scripts/align.sh

# The link can also be given an arbitrary name/path:
ln -s ~/share/scripts/align.sh align_from-shared.sh
Note

It’s best to always use an absolute path for the source file(s).

The $PWD environment variable can come in handy to do so:

ln -s $PWD/shared-scripts/align.sh project1/scripts/

Multiple files

  • Link to multiple files in a directory at once:

    ln -s $PWD/shared_scripts/* project1/scripts/

  • Link to a directory:

    ln -s $PWD/shared_scripts/ project1/scripts/
ln -s $PWD/shared_scripts/ project1/scripts/ln-shared-scripts
Warning

Be careful when linking to directories: you are creating
a point of entry to the original dir.

Therefore, even if you enter via the symlink, you are interacting with the original files:

Do not use rm -rf symlink-to-dir!

Instead, rm symlink-to-dir or unlink symlink-to-dir will only remove the link.

Creating symbolic links: practice

Create a symbolic link in your $HOME dir that points to your personal dir in the project dir (/fs/ess/PAS1855/users/$USER).

If you don’t provide a name for the link, it will be your username (why?), which is not particularly informative about its destination. Therefore, give it a name that makes sense to you, like PLNTPTH8300-SP21 or pracs-sp21.

ln -s /fs/ess/PAS1855/users/$USER $HOME/PLNTPTH8300-SP21

What will happen if you do rm -rf $HOME/PLNTPTH8300-SP21?

The content of the original dir will be removed.

10.3 More conditional file processing

To remove all empty files in a directory:

for file in *; do
>   if [ ! -s "$file" ]
>   then
>      echo rm "$file"
>   fi
> done

An overview of file test operators that can be used in this way:

Operator Returns true if:
-s file is not zero size
-e file exists
-f file is a regular file (e.g. not a directory)
-d file is a directory
-h file is a symbolic link
-r/ -w / -x file has read/write/execute permissions
Note

Constructions similar to the one above can be applied to many tasks,
though this particular task can be achieved much more concisely using find:

find . -size 0 -delete
